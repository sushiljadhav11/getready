1. Introduction to Operating Systems

An Operating System (OS) is the foundational layer of software that manages computer hardware and software resources, providing common services for computer programs. Fundamentally, an OS acts as an intermediary between the user of a computer and the computer hardware. Without an OS, every program would require its own massive logic completely outlining how to render pixels to the screen, spin hard drives, and execute clock cycles on the CPU.

Primary Goals:

  • Convenience: Makes a computer system convenient to use.
  • Efficiency: Manages system resources (CPU cache, memory, I/O devices) efficiently.
  • Ability to Evolve: Permits effective development, testing, and introduction of new system functions.

System Calls: Programs communicate with the OS entirely via System Calls (e.g., trying to read a file triggers an I/O system call passing control to kernel-level execution). This causes a Context Switch from User Mode to Kernel Mode.

2. Process & Thread Management

Process: A program in execution. It is an active entity natively isolated by the OS context. It spans multiple distinct memory zones:

  • Text Section: The compiled program code.
  • Data Section: Global variables.
  • Heap: Memory dynamically allocated at run time (e.g., via malloc()).
  • Stack: Temporary data directly used for function execution (local variables, return addresses).

Thread: A logical execution unit within a single process. A process can heavily parallelize via multiple threads. Critically, threads share the Data, Heap, and Text sections with each other, but they possess their own individual Registers and Stacks.

Thread swapping is extraordinarily fast compared to Process swapping because no heavy memory barriers or cache flushes are logically required.

3. Process Scheduling Algorithms

Scheduling dictates exactly how the CPU operates between hundreds of competing processes natively operating concurrently.

  • First Come, First Served (FCFS): The simplest algorithm natively executing processes strictly in the order they arrive. Notorious for the Convoy Effect, where a massive process blocks thousands of microscopic ones behind it.
  • Shortest Job First (SJF): Calculates the runtime required for all queued operations and executes the shortest one natively. Mathematically guarantees the lowest average waiting time, provided process execution time can be correctly predicted.
  • Round Robin: Designed exclusively for Time-Sharing systems. Every process is given a native CPU Time Quantum (e.g., 10ms). Once the quantum drastically expires, the process is inherently preempted and placed at the back of the total Ready Queue.
  • Multilevel Feedback Queue: The most complex, maintaining dozens of queues. If a process exhausts its quantum heavily without executing I/O, it gets pushed strictly to a lower-priority queue. Only processes displaying lightning-fast I/O behaviors remain natively at the top queues.

4. Concurrency & Synchronization

When massive multi-threaded programs execute, immense synchronization structures are drastically required to avoid unpredictable Race Conditions (where parallel modifications permanently corrupt the data).

Critical Section: A localized segment of code accessing heavily shared variables natively. Only one process mathematically is allowed within the critical section at any given time (Mutual Exclusion).

  • Mutex (Mutual Exclusion): A basic binary locking mechanism natively deployed entirely for local thread locking. Once a thread captures the lock, no other thread mathematically enters the critical boundary.
  • Semaphores: Integer variables fundamentally signaling resource availability drastically. A Counting Semaphore initializes statically to the total available pooled resources natively. Operations include Wait (P) and Signal (V).
  • Monitors: Natively higher-level synchronization abstractions statically programmed into robust languages like Java intrinsically to streamline thread queue execution logic dynamically.
